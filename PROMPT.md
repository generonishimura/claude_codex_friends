# タスク: 使い勝手の問題を順番に修正する

以下の問題を優先度順に修正してください。各修正はブランチ上でコミットしてください。

## 修正対象の問題リスト（優先度順）

1. ハードコード値の散在・重複を解消 - CLI_STARTUP_DELAY_MS(3箇所重複)、セッション名(2箇所重複)、チャンクサイズ等をconfig/定数に集約
2. 一時ファイルのクリーンアップ - /tmp/ccf/ のファイルをループ終了時に削除する処理を追加
3. エラーからのリカバリー（リトライロジック） - タイムアウトやキャプチャ失敗時に設定回数リトライする仕組みを追加
4. APPROVED判定の改善 - isApproved()のパターンを拡充し、LLMの多様な表現（LGTM, Approved with suggestions等）に対応
5. コード抽出の信頼性向上 - extractCodeFromResponse()の改善（最長ブロック選択ロジックの見直し等）
6. プロンプトのカスタマイズ対応 - 生成・レビュー・修正プロンプトをユーザーが設定ファイルやCLIオプションで上書き可能にする
7. Auto mode後のセッション管理改善 - セッションのライフサイクルを明確化（--keep-sessionフラグ等）
8. REPLモードの機能拡充 - 履歴表示、前回結果の再利用、ページネーション等
9. 出力結果の保存 - イテレーション履歴・レビュー内容をJSON/ログファイルとして保存するオプション追加
10. CLI応答検知の堅牢化 - 完了判定パターンの拡充、動的タイムアウト等

## ルール

- mainブランチに直接コミット禁止。作業開始前にブランチを作成すること
- ブランチがまだなければ refactor/improve-usability で作成
- 各問題の修正ごとにコミット（アトミックコミット原則）
- TDD: ドメインロジックの変更にはテストを先に書く
- 型安全性を維持（any禁止）
- Result型パターンを維持
- 修正後は npm run typecheck と npm test で確認
- 上のチェックリストを更新して進捗を記録する
